## 三、四大核心流程

### 流程1：评审批次管理 (EvaluationBatchService)

| 状态 | 操作方法 | 说明 |
|------|----------|------|
| NOT_STARTED | publishBatch() | 发布批次，开启申请 |
| APPLYING | startReview() | 开始评审 |
| REVIEWING | startPublicity() | 开始公示 |
| PUBLICITY | closeBatch() | 完成评定 |

### 流程2：评分计算 (EvaluationCalculationService)

**四个评分维度及权重**：

| 维度 | 权重 | 计分方式 |
|------|------|----------|
| 课程成绩 (courseScore) | 40% | 平均分/或AHP法 |
| 科研成果 (researchScore) | 30% | 论文+专利+项目加权和 |
| 竞赛获奖 (competitionScore) | 20% | 竞赛级别×获奖等级 |
| 综合素质 (qualityScore) | 10% | 德育表现 |

**科研成果内部计算**：
- 论文：期刊级别×作者排名系数（第1作者1.0，第2作者0.7...）
- 专利：专利类型×申请人排名系数（第1申请人1.0，第2作者0.6...）
- 项目：项目类型×成员排名系数

**总分公式**：
```
totalScore = courseScore×0.4 + researchScore×0.3 + competitionScore×0.2 + qualityScore×0.1
```

### 流程3：排名生成 (EvaluationRankService)

按两个维度排名：
- **院系排名** (departmentRank)：在同一院系内的排名
- **专业排名** (majorRank)：在同一专业内的排名

算法：按总分降序排列，依次分配排名

### 流程4：奖学金分配 (AwardAllocationService)

**奖项等级与默认比例**：

| 等级 | 名称 | 默认比例 | 默认金额 |
|------|------|----------|----------|
| 1 | 特等奖学金 | 5% | 10,000元 |
| 2 | 一等奖学金 | 10% | 5,000元 |
| 3 | 二等奖学金 | 20% | 3,000元 |
| 4 | 三等奖学金 | 30% | 1,000元 |
| 5 | 未获奖 | 35% | 0元 |

分配算法：按总分排名顺序，根据比例分配奖项

## 四、学生申请流程

```
学生提交申请
     ↓
关联成果（论文/专利/项目/竞赛，只能选审核通过的）
     ↓
填写自我评价
     ↓
提交申请 (status=1 已提交)
     ↓
┌─ 导师审核 (2→3通过 或 4不通过)
│     ↓
└─ 院系审核 (5→6不通过 或 7通过)
          ↓
      审核通过 (status=7)
          ↓
   等待系统评分计算
          ↓
   查看评定结果
     ↓
   （可选）提起异议申诉
```

### 9.3 两级审核机制

**实际只实现了两级审核**（非三级）：

| 审核阶段 | 用户类型 | 说明 |
|---------|---------|------|
| stage=1 | 导师 (type=2) | 只能审核自己指导的学生 |
| stage=2 | 院系管理员 (type=3) | 院系级审核，通过后直接APPROVED |

### 9.4 导师审核关键逻辑

```java
// ScholarshipApplicationServiceImpl.java
// 导师只能审核自己指导的学生
if (reviewStage == 1 && student.getTutorId() != null
    && !student.getTutorId().equals(reviewerId)) {
    throw new BusinessException("无权审核该申请，只能审核自己指导学生的申请");
}
```

### 9.5 审核意见转换 (ReviewOpinionEnum)

| 意见关键词 | 转换结果 |
|-----------|---------|
| 通过/同意/Agree | true (通过) |
| 不通过/拒绝/不同意/Disagree | false (不通过) |
| 其他 | null (待定) |

### 9.6 审核记录表 (review_record)

| 字段 | 说明 |
|------|------|
| application_id | 关联的申请ID |

## 十、异议申诉流程详解

### 10.1 申诉实体 (ResultAppeal)

| 字段 | 说明 |
|------|------|
| result_id | 关联的评定结果ID |
| appeal_type | 异议类型 |
| appeal_title | 异议标题 |
| appeal_reason | 异议原因 |
| appeal_content | 异议详细内容 |
| appeal_status | 申诉状态 (1=待处理, 2=处理中, 3=已处理) |
| handle_opinion | 处理意见 |
| handler_id | 处理人ID |

### 10.2 申诉状态枚举 (AppealStatusEnum)

| 状态码 | 状态名 | 说明 |
|--------|--------|------|
| 1 | PENDING | 待处理 |
| 2 | PROCESSING | 处理中 |
| 3 | PROCESSED | 已处理 |

### 10.3 申诉流程图

```
学生端:                                    管理端:
─────────────────                           ─────────────────
                                            ADMIN 查看申诉列表
                                            GET /result-appeal/page
┌─────────────┐    POST /result-appeal       │
│ 学生提出异议 │ ───────────────────────────►│ 管理员处理申诉
│ (公示中状态)│    resultId + reason +       │ PUT /result-appeal/handle/{id}
└─────────────┘    content                   └──────────────────────────┘
      │                                             │
      ▼                                             ▼
┌─────────────────────┐                    ┌──────────────────────────┐
│ ResultAppeal 创建    │                    │ 更新 appealStatus = 3    │
│ appealStatus = 1    │                    │ 填写 handleOpinion       │
└─────────────────────┘                    └──────────────────────────┘
```

### 10.4 申诉提交条件

- 结果状态必须为**公示中** (resultStatus=1)
- 只能对**本人**的评定结果提交申诉
- 申诉时需填写：异议理由 + 详细说明

### 10.5 申诉处理后不会自动修改评定结果

**重要发现**：管理员处理申诉后：
- 只更新 `result_appeal` 表状态为 PROCESSED
- **不会**自动修改 `evaluation_result` 的 `awardLevel`
- 如需调整奖项，管理员需手动在 Results.vue 中点击"调整"按钮
