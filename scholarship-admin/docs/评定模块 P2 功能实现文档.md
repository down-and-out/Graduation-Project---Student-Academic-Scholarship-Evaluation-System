## 二、详细功能说明

### 2.1 批次状态流转增强

#### 2.1.1 批次状态定义

| 状态码 | 名称 | 说明 |
|--------|------|------|
| 1 | 未开始 | 批次已发布，尚未开始申请 |
| 2 | 申请中 | 学生可以提交申请 |
| 3 | 评审中 | 导师/院系进行审核 |
| 4 | 公示中 | 评审结果公示期 |
| 5 | 已完成 | 批次结束 |

#### 2.1.2 新增 API 接口

| 接口路径 | 方法 | 权限 | 功能 |
|---------|------|------|------|
| `/evaluation-batch/publish/{id}` | PUT | ADMIN | 发布批次（设为未开始） |
| `/evaluation-batch/close/{id}` | PUT | ADMIN | 结束批次（设为已完成） |
| `/evaluation-batch/start-review/{id}` | PUT | ADMIN | 开始评审（申请中→评审中） |
| `/evaluation-batch/start-publicity/{id}` | PUT | ADMIN | 开始公示（评审中→公示中） |

#### 2.1.3 状态流转图

```
                    ┌─────────────┐
                    │  管理员创建  │
                    └──────┬──────┘
                           ↓
                    ┌─────────────┐
              ┌─────│  1-未开始   │
              │     └──────┬──────┘
              │            │ publish
              │            ↓
              │     ┌─────────────┐
              │     │  2-申请中   │◄─────┐
              │     └──────┬──────┘      │
              │            │ start       │ 关闭
              │            ↓             │
              │     ┌─────────────┐     │
              │     │  3-评审中   │─────┘
              │     └──────┬──────┘
              │            │ start-publicity
              │            ↓
              │     ┌─────────────┐
              │     │  4-公示中   │
              │     └──────┬──────┘
              │            │ close
              │            ↓
              │     ┌─────────────┐
              └────►│  5-已完成   │
                    └─────────────┘
```

#### 2.1.4 请求示例

**发布批次：**
```bash
PUT /evaluation-batch/publish/1
Authorization: Bearer {token}
```

**开始评审：**
```bash
PUT /evaluation-batch/start-review/1
Authorization: Bearer {token}
```

**开始公示：**
```bash
PUT /evaluation-batch/start-publicity/1
Authorization: Bearer {token}
```

**结束批次：**
```bash
PUT /evaluation-batch/close/1
Authorization: Bearer {token}
```

### 2.2 Excel 导出功能

#### 2.2.1 依赖配置

在 `pom.xml` 中添加 EasyExcel 依赖：

```xml
<!-- EasyExcel Excel 导出 -->
<dependency>
    <groupId>com.alibaba</groupId>
    <artifactId>easyexcel</artifactId>
    <version>3.3.4</version>
</dependency>
```

#### 2.2.2 API 接口

| 接口路径 | 方法 | 权限 | 功能 |
|---------|------|------|------|
| `/evaluation-result/export` | GET | ADMIN | 导出评定结果为 Excel |

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `batchId` | Long | 否 | 批次 ID，不传则导出所有 |

#### 2.2.3 导出文件格式

**Excel 列说明：**

| 列名 | 字段 | 说明 |
|------|------|------|
| 序号 | index | 行号 |
| 学号 | studentNo | 学生学号 |
| 姓名 | studentName | 学生姓名 |
| 院系 | department | 所在院系 |
| 专业 | major | 所学专业 |
| 课程成绩 | courseScore | 课程成绩分数 |
| 科研成果 | researchScore | 科研成果分数 |
| 竞赛获奖 | competitionScore | 竞赛获奖分数 |
| 综合素质 | qualityScore | 综合素质分数 |
| 总分 | totalScore | 加权总分 |
| 院系排名 | departmentRank | 院系内排名 |
| 专业排名 | majorRank | 专业内排名 |
| 获奖等级 | awardLevelName | 特等/一等/二等/三等/未获奖 |
| 奖学金金额 | awardAmount | 奖金数额 |
| 结果状态 | resultStatusName | 公示中/已确定/有异议 |

#### 2.2.4 请求示例

**导出某批次结果：**
```bash
GET /evaluation-result/export?batchId=1
Authorization: Bearer {token}
```

**导出所有结果：**
```bash
GET /evaluation-result/export
Authorization: Bearer {token}
```

#### 2.2.5 响应

- **Content-Type**: `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
- **Content-Disposition**: `attachment;filename=评定结果.xlsx`

#### 2.3.1 新增 API 接口

| 接口路径 | 方法 | 权限 | 功能 |

## 四、批次状态流转说明

### 4.1 完整流程

```
1. 管理员创建批次
   → 默认状态：1-未开始

2. 管理员发布批次
   → PUT /evaluation-batch/publish/{id}
   → 状态：1-未开始

3. 管理员开启申请
   → PUT /evaluation-batch/start/{id}
   → 状态：2-申请中
   → 学生可以提交申请

4. 申请截止，开始评审
   → PUT /evaluation-batch/start-review/{id}
   → 状态：3-评审中
   → 导师/院系审核申请

5. 评审完成，开始公示
   → PUT /evaluation-batch/start-publicity/{id}
   → 状态：4-公示中
   → 学生可以查看结果并提出异议

6. 公示期满，结束批次
   → PUT /evaluation-batch/close/{id}
   → 状态：5-已完成
```

### 4.2 状态检查建议

在关键业务节点建议进行状态检查：

```java
// 学生提交申请时检查批次状态
EvaluationBatch batch = evaluationBatchService.getById(batchId);
if (!batch.getBatchStatus().equals(2)) {
    return Result.error("当前批次不在申请中");
}

// 导师审核时检查批次状态
if (!batch.getBatchStatus().equals(3)) {
    return Result.error("当前批次不在评审中");
}
```

## 五、Excel 导出实现细节

### 5.1 导出 VO 注解说明

```java
@Data
@ColumnWidth(15)        // 列宽
@HeadRowHeight(20)      // 表头行高
public class EvaluationResultExportVO {

    @ExcelProperty("序号")
    @ColumnWidth(10)
    private Integer index;

    @ExcelProperty("学号")
    private String studentNo;

    // ... 其他字段
}
```

### 5.2 导出逻辑

```java
// 1. 查询数据
List<EvaluationResult> results = evaluationResultService.list(wrapper);

// 2. 转换为导出 VO
List<EvaluationResultExportVO> exportData = convertToExportVO(results);

// 3. 设置响应头
response.setContentType("application/vnd.openxmlformats-officedocument.spreadsheetml.sheet");
response.setHeader("Content-Disposition", "attachment;filename=评定结果.xlsx");

// 4. 使用 EasyExcel 写入
EasyExcel.write(response.getOutputStream(), EvaluationResultExportVO.class)
    .sheet("评定结果")
    .doWrite(exportData);
```

## 六、快速开始

### 6.1 批次状态流转操作

```bash
# 1. 创建批次后发布
PUT /evaluation-batch/publish/1

# 2. 开启申请
PUT /evaluation-batch/start/1

# 3. 学生提交申请
POST /application/submit

# 4. 开始评审
PUT /evaluation-batch/start-review/1

# 5. 导师审核
PUT /application/review/{id}

# 6. 开始公示
PUT /evaluation-batch/start-publicity/1

# 7. 一键评定（计算 + 排名 + 奖项）
POST /evaluation-result/evaluate/1

# 8. 结束批次
PUT /evaluation-batch/close/1
```
