### 2.2 评定计算服务

#### 2.2.1 服务接口

```java
public interface EvaluationCalculationService {
    // 计算单个申请的总分
    EvaluationResult calculateApplication(ScholarshipApplication application);

    // 批量计算某批次所有申请
    Map<Long, EvaluationResult> calculateBatchApplications(Long batchId);

    // 分类分数计算
    BigDecimal calculateCourseScore(Long studentId, Long batchId);
    BigDecimal calculateResearchScore(Long studentId, Long batchId);
    BigDecimal calculateCompetitionScore(Long studentId, Long batchId);
    BigDecimal calculateQualityScore(Long studentId, Long batchId);

    // 总分计算（支持加权）
    BigDecimal calculateTotalScore(BigDecimal courseScore, BigDecimal researchScore,
                                   BigDecimal competitionScore, BigDecimal qualityScore);
}
```

#### 2.2.2 分数计算逻辑

**1. 科研成果分数计算：**
- 论文分数 = 基础分 × 作者排名系数
  - 第一作者：100%
  - 第二作者：70%
  - 第三作者：50%
- 专利分数 = 基础分 × 发明人排名系数
  - 第一发明人：100%
  - 第二发明人：60%
  - 第三发明人：40%
- 项目分数 = 基础分 × 成员排名系数
  - 项目负责人：100%
  - 核心成员：70%
  - 一般成员：50%

**2. 竞赛获奖分数计算：**
- 根据竞赛级别和获奖等级匹配规则
- 团队项目按成员排名分配分数
  - 队长：100%
  - 副队长：80%
  - 队员：60%

**3. 总分计算公式：**
```
总分 = 课程成绩 × 0.4 + 科研分数 × 0.3 + 竞赛分数 × 0.2 + 综测分数 × 0.1
```

### 2.4 奖项分配服务

#### 2.4.1 服务接口

```java
public interface AwardAllocationService {
    // 批量分配奖项
    AwardAllocationResult allocateAwards(Long batchId);

    // 为单个评定结果分配奖项
    boolean allocateAward(EvaluationResult result, Map<Integer, AwardQuota> quotaConfig);

    // 获取奖项等级名称
    String getAwardLevelName(Integer awardLevel);

    // 根据排名确定奖项等级
    Integer determineAwardLevel(Integer rank, Integer total, Map<Integer, AwardQuota> quotaConfig);

    // 计算奖学金金额
    BigDecimal calculateAwardAmount(Integer awardLevel, Long batchId);
}
```

#### 2.4.2 奖项等级定义

| awardLevel | 名称 | 默认比例 | 默认金额 |
|------------|------|----------|---------|
| 1 | 特等奖学金 | 5% | ¥10,000 |
| 2 | 一等奖学金 | 10% | ¥5,000 |
| 3 | 二等奖学金 | 20% | ¥3,000 |
| 4 | 三等奖学金 | 30% | ¥1,000 |
| 5 | 未获奖 | - | ¥0 |

#### 2.4.3 分配逻辑

```java
// 名额计算（向上取整）
特等奖名额 = ceil(总人数 × 5%)
一等奖名额 = ceil(总人数 × 10%)
二等奖名额 = ceil(总人数 × 20%)
三等奖名额 = ceil(总人数 × 30%)

// 按排名依次分配
if (排名 <= 特等奖名额) → 特等奖
else if (排名 <= 特等奖 + 一等奖) → 一等奖
else if (排名 <= 特等奖 + 一等奖 + 二等奖) → 二等奖
else if (排名 <= 特等奖 + 一等奖 + 二等奖 + 三等奖) → 三等奖
else → 未获奖
```

#### 2.4.4 内部类

**AwardQuota（奖项配额配置）：**
```java
class AwardQuota {
    private Integer awardLevel;      // 奖项等级
    private Double ratio;            // 配额比例
    private BigDecimal amount;       // 单项金额
    private Integer maxCount;        // 最大名额
}
```

**AwardAllocationResult（分配结果统计）：**
```java
class AwardAllocationResult {
    private int specialCount;        // 特等奖人数
    private int firstCount;          // 一等奖人数
    private int secondCount;         // 二等奖人数
    private int thirdCount;          // 三等奖人数
    private int noAwardCount;        // 未获奖人数
    private BigDecimal totalAmount;  // 总金额
}
```

## 七、常见问题

### Q1: 为什么课程成绩和综合素质分数返回 0？

**A:** 这两个模块的计算逻辑需要对接实际的成绩表和德育表现表，目前在代码中标记为 TODO。请根据学校实际情况补充相关数据源。

### Q2: 如何修改奖项配额比例？

**A:** 修改 `AwardAllocationServiceImpl` 中的常量：
```java
private static final double SPECIAL_RATIO = 0.05;  // 特等奖比例
private static final double FIRST_RATIO = 0.10;    // 一等奖比例
// ...
```

### Q3: 如何修改默认权重？

**A:** 修改 `EvaluationCalculationServiceImpl.calculateTotalScore()` 方法中的权重系数。

### Q4: 评定过程中途失败怎么办？

**A:** 所有服务方法都使用了 `@Transactional` 事务注解，失败会自动回滚。重新执行即可。
