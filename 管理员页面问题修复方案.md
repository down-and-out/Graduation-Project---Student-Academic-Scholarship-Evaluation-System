# 导师页相关 7 个问题修复方案

## 执行状态：全部完成 ✅

后端编译 `mvn clean compile` 通过，前端类型检查 `vue-tsc --noEmit` 无错误。

## Summary
本次修复按"先堵权限漏洞，再收敛状态机，最后优化导师页查询链路"的顺序处理，目标是不改现有核心业务入口和前端页面信息架构，只做兼容式修复。

默认实现策略：
- 保持现有页面路由和主要接口路径不变。
- 权限问题统一复用现有 `DataScopeHelper`，避免各 Controller 再写一套角色判断。
- 论文"可用于导师统计/学生申报"的有效通过状态，按现有业务文档推断为"导师通过即可"，因此统计口径改为包含 `status=1`，并兼容保留 `status=2` 作为更高阶段通过状态。
- 年级筛选改为后端提供专用选项接口，移除前端 `size=1000` 的全量探测请求。
