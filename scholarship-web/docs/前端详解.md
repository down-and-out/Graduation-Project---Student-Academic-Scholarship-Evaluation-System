## 1. 项目概述

**项目名称**: 研究生学业奖学金评定系统（Scholarship Evaluation System）

**项目描述**: 这是一个用于管理研究生学业奖学金评定的 Web 应用前端项目，支持三种用户角色：
- **研究生**: 查看个人信息、管理科研成果、提交奖学金申请、查询评定结果
- **导师**: 审核学生科研成果、管理指导学生
- **管理员**: 用户管理、规则管理、评定管理、结果管理、系统设置

### 5.5 composables/ - 组合式函数（Hooks）

- 查询/重置处理

**返回值**:
```typescript
{
  loading: Ref<boolean>       // 加载状态
  tableData: T[]              // 表格数据
  total: Ref<number>          // 总数
  queryParams: Object         // 查询参数
  loadData: Function          // 加载数据
  handleQuery: Function       // 查询
  handleReset: Function       // 重置
  handleSizeChange: Function  // 每页大小变化
  handleCurrentChange: Function // 当前页变化
  refresh: Function           // 刷新
}
```

**使用示例**:
```typescript
const { loading, tableData, total, handleQuery, handleReset } = useTable(
  (params) => getStudentPage(params),
  { pageSize: 10, immediate: true }
)
```

### 5.7 router/ - 路由配置

#### `index.ts` - 路由定义和守卫

**路由结构**:

```
/                          → 重定向到 /login
/login                     → 登录页
/app                       → 主应用布局（需认证）
  /app/dashboard           → 首页
  /app/student/*           → 研究生菜单（需学生角色）
    /app/student/profile   → 个人信息
    /app/student/achievements → 科研成果
    /app/student/application → 奖学金申请
    /app/student/result    → 评定结果
  /app/tutor/*             → 导师菜单（需导师角色）
    /app/tutor/review      → 科研成果审核
    /app/tutor/students    → 指导学生
  /app/admin/*             → 管理员菜单（需管理员角色）
    /app/admin/users       → 用户管理
    /app/admin/students    → 研究生信息管理
    /app/admin/rules       → 评分规则管理
    /app/admin/evaluation  → 评定管理
    /app/admin/results     → 结果管理
    /app/admin/system      → 系统设置
/:pathMatch(.*)*           → 404 页面
```

**导航守卫功能**:
1. **认证检查**: 验证 token 和用户信息
2. **角色权限验证**: 检查用户角色是否匹配路由要求
3. **页面标题设置**: 动态设置 document.title
4. **未登录重定向**: 携带 redirect 参数跳转到登录页

**核心代码**:
```typescript
router.beforeEach((to, from, next) => {
  // 1. 设置页面标题
  // 2. 检查是否已登录（token + 用户信息）
  // 3. 需要认证的路由 → 未登录则跳转登录页
  // 4. 需要特定角色的路由 → 验证角色权限
  // 5. 允许访问
})
```

### 5.10 utils/ - 工具函数

#### `request.ts` - Axios 请求封装

**功能**:
- 创建 Axios 实例（baseURL, timeout）
- **请求拦截器**:
  - 添加 Authorization header（Bearer token）
  - 添加请求 ID（防重复提交）
- **响应拦截器**:
  - 统一错误处理
  - Token 过期自动登出并跳转登录页
  - HTTP 状态码错误处理
- **网络错误处理**: 区分请求错误、响应错误、配置错误

**Token 错误处理流程**:
```
Token 过期/无效 → handleLogoutAndRedirect()
                → 清除用户状态
                → 调用 logout API
                → 跳转 /login
```

#### `secureStorage.ts` - 安全存储

**功能**:
- **tokenStore**: Token 内存存储（session 级别）
- **userInfoStore**: 用户信息存储（XOR 混淆 + sessionStorage）
- **安全清除**: 同时清除内存和存储

**安全策略**:
- Token 不持久化，关闭标签页即清除
- 用户信息 XOR 混淆，防止 XSS 直接读取

#### `rsa.ts` - RSA 加密工具

**功能**:
- 使用 JSEncrypt 库进行 RSA 加密
- 从环境变量读取公钥
- 密码加密传输

**使用场景**:
- 登录时加密密码
- 敏感数据传输

#### `helpers.ts` - 通用工具函数

**提供函数**:
- `generateUUID()`: 生成 UUID
- `debounce(fn, delay)`: 防抖函数
- `throttle(fn, interval)`: 节流函数
- `formatDate(date, format)`: 日期格式化
- `isValidUsername(str)`: 用户名验证
- `isValidPhone(str)`: 手机号验证
- `isValidEmail(str)`: 邮箱验证
- `isValidIdCard(str)`: 身份证验证
- `isValidPassword(str)`: 密码验证
- `formatFileSize(bytes)`: 文件大小格式化
- `deepClone(obj)`: 深拷贝

#### `errorHandler.ts` - 全局错误处理

**功能**:

- 错误上报（可扩展发送到服务器）

## 6. 核心功能模块

### 6.1 认证与授权

**登录流程**:
```
1. 用户输入用户名密码
2. 使用 RSA 公钥加密密码
3. 调用 login API
4. 成功后存储 token 和用户信息
5. 跳转到 Dashboard
```

**权限控制**:
- 路由守卫验证 token 和用户信息
- 基于角色的菜单显示
- 路由级别的 role 元数据验证

### 6.2 数据流

```
用户交互
    ↓
View/Component
    ↓
Composables (useTable, useDialogForm)
    ↓
API 层
    ↓
request.ts (Axios 封装)
    ↓
后端 API
    ↓
响应数据
    ↓
Stores (Pinia)
    ↓
View 更新
```

### 6.3 安全特性

1. **RSA 密码加密**: 登录密码使用 RSA 公钥加密后传输
2. **Token 安全存储**: 使用 sessionStorage（关闭标签页自动清除）
3. **用户信息混淆**: XOR 混淆防止 XSS 直接读取
4. **CSP 策略**: HTML 中配置内容安全策略
5. **路由守卫**: 基于角色的访问控制
6. **请求 ID**: 防止重复提交

## 7. 页面路由系统

### 7.1 路由守卫流程

```
路由跳转触发 beforeEach
        ↓
设置页面标题
        ↓
检查是否需要认证
        ↓
    ┌───┴───┐
    ↓       ↓
  需要    不需要
    ↓       ↓
检查登录    直接放行
    ↓
未登录 → 清除状态 → 跳转登录页
    ↓
已登录 → 检查角色权限
    ↓
角色匹配 → 允许访问
角色不匹配 → 提示无权访问 → 跳转首页
```

### 7.2 路由元数据

```typescript
interface RouteMeta {
  title?: string       // 页面标题
  icon?: string        // 菜单图标
  requiresAuth?: boolean  // 是否需要认证
  role?: 'student' | 'tutor' | 'admin'  // 角色要求
}
```

## 8. 架构设计模式

### 8.1 分层架构

```
┌─────────────────────────────────────┐
│           Views (页面层)             │
│  Login, Dashboard, Profile...       │
├─────────────────────────────────────┤
│        Components (组件层)           │
│  ErrorBoundary, AppLayout...        │
├─────────────────────────────────────┤
│       Composables (组合式函数)       │
│  useTable, useDialogForm...         │
├─────────────────────────────────────┤
│         Stores (状态管理)            │
│  user (Pinia store)                 │
├─────────────────────────────────────┤
│          API (接口层)                │
│  auth, student, application...      │
├─────────────────────────────────────┤
│        Utils (工具层)                │
│  request, secureStorage, rsa...     │
├─────────────────────────────────────┤
│       Constants (常量层)             │
│  resultCode, index                  │
└─────────────────────────────────────┘
```

### 8.2 设计模式

1. **组合式 API (Composition API)**: 使用 `<script setup>` 语法
2. **单向数据流**: 数据从 API → Store → View 单向流动
3. **依赖注入**: Pinia stores、Vue Router 通过 provide/inject 注入
4. **单例模式**: ErrorHandler 使用单例模式
5. **工厂模式**: Axios 实例创建、Pinia store 创建

### 8.3 代码规范

- **TypeScript**: 严格类型检查，所有函数和变量都有类型注解
- **JSDoc**: 关键函数和变量有详细的文档注释
- **命名规范**:
  - 组件：PascalCase（如 `ErrorBoundary.vue`）
  - 函数：camelCase（如 `handleLogin`）
  - 常量：UPPER_SNAKE_CASE（如 `USER_ROLE`）
  - 类型：PascalCase（如 `UserInfo`）

### 新增页面步骤
